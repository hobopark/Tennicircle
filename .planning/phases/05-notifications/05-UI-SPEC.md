## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge offsets, inline padding |
| sm | 8px | Compact row padding, icon-to-label gap |
| md | 16px | Default card padding, list item padding |
| lg | 24px | Section padding, page horizontal inset (`px-6`) |
| xl | 32px | Layout gaps between sections |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Bottom nav clearance (`pb-16`) |

Exceptions:
- Touch targets minimum 44px height for notification rows (accessibility)
- Bottom nav is 64px fixed height (`h-16`); page content must clear it with `pb-20`
- Unread badge: 20px diameter (`w-5 h-5`), 4px offset from icon corner

## Color

Source: globals.css `:root` block

| Role | Token | HSL | Usage |
|------|-------|-----|-------|
| Dominant (60%) | `--background` | `hsl(60 10% 98%)` | Page background, notification feed background |
| Secondary (30%) | `--card` | `hsl(0 0% 100%)` | Notification row surfaces, popover container |
| Accent (10%) | `--primary` | `hsl(152 65% 42%)` | Bell icon active state, unread badge, unread row left border |
| Muted surface | `--muted` | `hsl(150 10% 94%)` | Inactive nav icons, read notification row background |
| Muted text | `--muted-foreground` | `hsl(160 10% 45%)` | Timestamp text, secondary notification body |
| Destructive | `--destructive` | `hsl(0 84.2% 60.2%)` | Not used in this phase (no destructive actions) |

Accent (`--primary`) reserved for:
1. Bell icon when active tab is selected in bottom nav
2. Unread badge count pill on bell icon
3. Unread indicator dot on individual notification rows
4. Left border accent stripe on unread notification cards

Secondary warm (`--secondary`, `hsl(45 100% 58%)`) is NOT used in notification UI — reserved for event/tournament type badges established in Phase 4.

## Notification Row Anatomy

Each notification row renders as a card-style list item with this structure:

```
[ avatar (32px) ] [ title (semibold, text-base) ]        [ timestamp (text-[10px] muted) ]
                   [ body copy (text-sm, muted) ]
```

Visual states:
- **Unread**: `bg-card` surface with `border-l-2 border-primary` left accent stripe; title at `font-semibold`
- **Read**: `bg-muted/40` surface, no left stripe; title at `font-normal`
- **Tap/active**: `active:scale-[0.98] transition-transform` — matches AceHub card press pattern

Row minimum height: 44px touch target. Actual rendered height will be ~56–64px with padding.

## Notification Types

Three types with distinct iconography:

| Type | Trigger | Icon | Icon color class |
|------|---------|------|-----------------|
| Session reminder (NOTF-01) | Cron job 24h before session | `CalendarDays` (lucide) | `text-primary` |
| Announcement (NOTF-02) | `createAnnouncement` server action | `Megaphone` (lucide) | `text-primary` |
| RSVP confirmation / waitlist promotion (NOTF-03) | `rsvpSession`, `rsvpEvent`, auto-promotion | `CheckCircle2` (lucide) | `text-primary` |

Type icons replace the avatar when no sender persona is relevant (e.g., system-generated reminders). For announcement notifications, the posting coach/admin's `InitialsAvatar` is shown instead.

## Navigation Integration

Bell tab added to `AppNav.tsx` for all roles (admin, coach, client):

```
href: /notifications
label: Notifications
icon: Bell (lucide, w-5 h-5)
roles: ['admin', 'coach', 'client']
```

Unread badge:
- Position: absolute, top-right corner of icon container, `-top-1 -right-1`
- Size: `w-5 h-5` (20px diameter)
- Style: `bg-primary text-primary-foreground text-[10px] font-bold rounded-full`
- Cap: shows "9+" when count exceeds 9
- Visibility: hidden when count is 0 (do not render empty badge)

Active tab pattern (from existing AppNav): `bg-primary text-primary-foreground p-1.5 rounded-xl transition-all duration-300 shadow-sm` on icon container.

## Copywriting Contract

| Element | Copy |
|---------|------|
| Page heading | Notifications |
| Mark all as read CTA | Mark all as read |
| Empty state heading | You're all caught up |
| Empty state body | New session reminders, announcements, and RSVP updates will appear here. |
| Notification — session reminder | Your session is tomorrow at {time} — {venue} |
| Notification — announcement | {coach/admin name} posted an announcement |
| Notification — RSVP confirmed | You're confirmed for {session/event name} on {date} |
| Notification — waitlist promoted | You've been moved off the waitlist for {session/event name} |
| Unread badge — overflow | 9+ |
| Loading state aria-label | Loading notifications |

Tone: Concise, factual, action-oriented. No exclamation marks except waitlist promotion (celebratory moment). No first-person from the app ("We confirmed you") — use passive or second-person ("You're confirmed").

## Deep-Link Behaviour

Each notification row is tappable and navigates to the relevant content:

| Notification type | Deep-link target |
|-------------------|-----------------|
| Session reminder | `/sessions/{sessionId}` |
| Announcement | `/events` (announcements tab) |
| RSVP confirmed — session | `/sessions/{sessionId}` |
| RSVP confirmed — event | `/events/{eventId}` |
| Waitlist promotion — session | `/sessions/{sessionId}` |
| Waitlist promotion — event | `/events/{eventId}` |

Tapping a notification marks it as read and navigates. Navigation uses Next.js `router.push()` from a client component.

## Interaction States

| State | Treatment |
|-------|-----------|
| Loading (initial fetch) | 3 skeleton rows at standard row height (56px each) with `Skeleton` component |
| Real-time new notification arrives | Row prepends to top of list with `initial={{ opacity: 0, y: -8 }} animate={{ opacity: 1, y: 0 }}` framer-motion entrance |
| Mark all as read (button tap) | Optimistic: all rows visually switch to read state immediately; server action confirms |
| Empty feed | Centered layout: `Bell` icon (w-12 h-12 text-muted-foreground/40) + heading + body copy |
| Realtime subscription error | Silent degradation — feed still works as a static list; no error banner needed |
