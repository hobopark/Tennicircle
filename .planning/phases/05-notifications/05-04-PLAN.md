**Step 1: Push the database schema**

Run the Supabase schema push. The project uses `supabase db push` to apply migrations.

```bash
npx supabase db push
```

If this requires interactive confirmation, try:
```bash
echo "y" | npx supabase db push
```

If the push requires `SUPABASE_ACCESS_TOKEN` for non-TTY:
```bash
SUPABASE_ACCESS_TOKEN=$SUPABASE_ACCESS_TOKEN npx supabase db push
```

If the push fails due to auth, flag for manual intervention (the user will need to run `supabase db push` manually or provide the access token).

**Step 2: Enable Realtime for notifications table**

The migration includes `alter publication supabase_realtime add table public.notifications`. If this doesn't take effect via migration, the user may need to enable Realtime for the `notifications` table in the Supabase Dashboard:
- Go to Supabase Dashboard -> Database -> Replication
- Ensure `notifications` table is in the `supabase_realtime` publication

**Step 4: Verify type checking**

Must pass clean.
