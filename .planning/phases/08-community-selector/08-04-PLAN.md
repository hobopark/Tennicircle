## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browse list -> Join request | User can only request join for communities they see (server filters already-joined) |
| Create community | Only admins see the + Create card; server action validates admin role |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-13 | Elevation of Privilege | createCommunity | mitigate | Server action checks if user is admin in ANY community. Non-admins cannot create communities. |
| T-08-14 | Spoofing | requestToJoin | mitigate | Server action uses auth.uid() from session, not client-provided user_id. RLS enforces user_id = auth.uid(). |
| T-08-15 | Denial of Service | Browse page load | accept | getBrowseCommunities fetches all non-joined communities. With small community count (MVP), this is fine. |
</threat_model>
