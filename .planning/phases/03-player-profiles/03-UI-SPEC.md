## Spacing Scale

All spacing uses the 8-point scale. Tailwind utility classes map directly; do not use arbitrary values unless listed as an exception.

| Token | Value | Tailwind | Usage |
|-------|-------|----------|-------|
| xs | 4px | `gap-1`, `p-1` | Badge padding, icon gaps |
| sm | 8px | `gap-2`, `p-2` | Inline element spacing, label-to-input gap |
| md | 16px | `gap-4`, `p-4` | Default card internal padding, field spacing |
| lg | 24px | `gap-6`, `p-6` | Section padding, stat card padding |
| xl | 32px | `gap-8`, `p-8` | Page section breaks |
| 2xl | 48px | `py-12` | Major layout gaps between profile sections |
| 3xl | 64px | `py-16` | Page top padding on desktop |

Exceptions:
- Avatar touch target: 44px minimum on mobile — use `min-h-[44px] min-w-[44px]` on upload trigger
- Wizard step indicators: 32px circle — `w-8 h-8`
- Progress note textarea: minimum height 80px — `min-h-20`
- Input height: 48px — `h-12` (matches Phase 1 LoginForm pattern)
- Page horizontal padding: 20px — `px-5` (matches AceHub home page pattern)
- Bottom nav height: 64px fixed — accounts for safe area inset on iOS

## Typography

| Role | Font | Size | Weight | Line Height | Tailwind |
|------|------|------|--------|-------------|---------|
| Display heading | Space Grotesk | 24px | 700 | 1.2 | `font-heading font-bold text-2xl leading-tight` |
| Stat value | Space Grotesk | 24px | 700 | 1.2 | `font-heading font-bold text-2xl` |
| Body / section heading / card title | Inter | 16px | 400 | 1.5 | `text-base` (headings add `font-heading font-bold`) |
| Label / helper / tag | Inter | 14px | 400 | 1.5 | `text-sm` |
| Micro label | Inter | 10px | 400 | 1.4 | `text-[10px]` |

Rules:
- Exactly 4 font sizes: 24px, 16px, 14px, 10px. Do not introduce any other size.
- Exactly 2 font weights: **400** (Inter body, labels, helpers) and **700** (Space Grotesk headings, bold labels). Do not use 500 or 600 anywhere.
- Use `font-heading font-bold` for all headings regardless of size (section headings, card titles, stat values, display headings all use Space Grotesk 700).
- Use default `font-sans` (Inter 400) for body text, labels, helper text, micro labels.
- Stat values (sessions attended count, etc.) use `font-heading font-bold text-2xl` — consolidated from former 20px `text-xl` row.
- Former `text-xs` (12px "Small / tag") rows are folded into `text-sm` (14px). Use `text-sm` for badge text, tags, and secondary labels.
- Stat labels below stat values: `text-[10px] text-muted-foreground uppercase tracking-wide` — no `font-medium`, weight 400 only.
- Bottom nav labels: `text-[10px]`, active state uses `font-bold` (700).
- Progress notes text (read-only): `text-sm text-muted-foreground`.
- Coach-assessed skill label: `text-sm font-bold text-foreground` — carries authority, not muted.
- "See all" section links: `text-sm text-primary`.

- Active wizard step number circle background

## Page and Surface Inventory

### `/profile/setup` — Profile Setup Wizard

**Surface:** Centered card, max-width 440px. Mobile: full-bleed, no shadow. Desktop: `bg-card rounded-3xl border border-border/50 shadow-[0_4px_24px_rgba(0,0,0,0.06)] p-8`. Full-bleed on mobile matches AceHub pattern.

**Step sequence (4 steps):**

| Step | Name | Fields |
|------|------|--------|
| 1 | Identity | Display name (required), bio (optional, textarea) |
| 2 | Contact | Phone number (optional), email (read-only, auto-populated from auth) |
| 3 | Avatar | Photo upload with square crop preview, or skip |
| 4 | Skill | Self-assessed skill level (3-option radio), UTR (optional number input) |

**Wizard navigation:**
- Step indicator: horizontal row of 4 numbered circles at top of card
  - Completed: `w-8 h-8 rounded-full bg-muted text-muted-foreground` with checkmark Lucide icon
  - Active: `w-8 h-8 rounded-full bg-accent text-accent-foreground font-heading font-bold` (accent reserved use)
  - Upcoming: `w-8 h-8 rounded-full bg-muted text-muted-foreground opacity-40`
  - `aria-current="step"` on active circle; `aria-label="Step {N} of 4: {name}"` on each
- Step connector: 1px `border-t border-border` horizontal line between circles, `flex-1`
- Back button: `text-sm text-muted-foreground hover:underline` — ghost text, left-aligned under card
- Next/Continue button: full-width primary, `h-12 rounded-2xl font-heading font-bold`
- Skip link on steps 2, 3, 4: `text-sm text-muted-foreground hover:underline` centered below primary button

**Card title:** `font-heading font-bold text-2xl` — "Set up your profile" (new) / "Edit your profile" (returning)

**Avatar step (step 3):**
- Upload trigger: 80px circle, `rounded-2xl border-2 border-dashed border-border bg-muted flex items-center justify-center cursor-pointer min-h-[44px] min-w-[44px]`; "+" icon `w-6 h-6 text-muted-foreground`
- After file selected: square crop preview rendered in same 80px area via canvas 1:1 crop
- File constraints: max 5MB, accept `image/jpeg, image/png, image/webp` — validate client-side
- Error shown inline below upload circle: `text-destructive text-sm mt-1`

**Initials avatar fallback (no photo):**
- 80px div, `rounded-2xl` (matches AceHub avatar shape — not fully circular)
- Background: `bg-gradient-to-br from-primary/30 to-primary/5`
- Text: first 2 initials, `font-heading font-bold text-base text-primary`
- `aria-label="Avatar for {display name}"`

**Skill level selector (step 4):**
- `<fieldset>` with visually hidden `<legend>` ("Your skill level")
- 3 styled radio options in a column: Beginner, Intermediate, Advanced
- Each option: `bg-muted rounded-2xl p-4 border border-border cursor-pointer transition-colors`
- Selected state: `border-primary bg-primary/5`
- Label: `text-base font-bold text-foreground font-heading`
- Sub-description per level (`text-sm text-muted-foreground` below label):
  - Beginner: "Just starting out or returning to the game"
  - Intermediate: "Consistent rallying and match experience"
  - Advanced: "Competitive play and tournament experience"

## Interaction Patterns

### Form Validation Timing

Follow Phase 1 pattern exactly:
- Server-side Zod validation on submit
- Field errors inline below each input: `text-destructive text-sm mt-1`
- On field `onChange`, clear that field's error locally
- No blur-time validation

### Loading States

- Wizard "Continue" / "Save" button: `<Loader2 className="animate-spin" size={16} />` left of text while pending; button disabled
- "Save note" and "Save assessment": same spinner pattern
- Avatar upload: replace upload circle content with spinner while uploading
- Lesson history list: skeleton — 3 placeholder rows `bg-muted animate-pulse rounded-3xl h-20`

### Entrance Animations

Use Framer Motion for phase-in on page load. Pattern from AceHub:
- Page sections: `initial={{ opacity: 0, y: -10 }} animate={{ opacity: 1, y: 0 }}`
- Stagger between cards: `transition={{ delay: index * 0.05 }}`
- Do NOT animate form fields — only container cards and section blocks

Framer Motion is not yet installed. The executor must add it: `npm install framer-motion`.

### Toast Feedback (Sonner)

| Action | Toast |
|--------|-------|
| Profile saved (wizard complete) | Success: "Profile saved" |
| Assessment updated | Success: "Assessment updated" |
| Note saved | Success: "Note saved" |
| Avatar upload failed (size) | Error: "Image must be under 5MB. Try a smaller file." |
| Avatar upload failed (type) | Error: "Please upload a JPEG, PNG, or WebP image." |
| Any server action error | Error: `{error message from action}` |

### Navigation After Wizard Complete

`router.push('/profile')` on successful final submit.

### Responsive Behaviour

- Wizard card: full-bleed on mobile (no `rounded-3xl`, no shadow), rounded + shadow on `sm:` and up
- Profile page: single column always; no grid layout changes
- Stats: `grid grid-cols-3` on all screen sizes
- Lesson history cards: full-width always
- Bottom nav: fixed `bottom-0`, `pb-[env(safe-area-inset-bottom)]` for iOS

### Bottom Navigation

Pattern from AceHub — applies to all authenticated pages:
- `fixed bottom-0 left-0 right-0 z-50`
- `bg-card/80 backdrop-blur-xl border-t border-border`
- `pb-[env(safe-area-inset-bottom)]`
- Active tab: `text-primary`, icon container `bg-primary/10 p-1.5 rounded-xl`
- Inactive tab: `text-muted-foreground`
- Label: `text-[10px]`, `font-bold` when active
- `AppNav.tsx` needs "Profile" link added: `{ href: '/profile', label: 'Profile', icon: User, roles: ['admin', 'coach', 'client'] }`

## Accessibility Contract

- All form inputs: `<Label htmlFor>` paired with input `id`
- Avatar images: `alt="{display name}'s avatar"` on uploaded photo; `alt=""` with `aria-label="Avatar for {display name}"` on initials/gradient fallback
- Skill level fieldset: `<fieldset>` + visually-hidden `<legend>` for screen reader group label
- Wizard step indicators: `aria-current="step"` on active step; `aria-label="Step {N} of 4: {step name}"` on each circle
- Lesson history: `<ul>` + `<li>` items, not divs
- Error messages: `aria-describedby` links input to its error `<p>`; `aria-invalid="true"` on errored inputs
- Progress note textareas: `aria-label="Progress note for {player name}"`
- Coach assessment inline form: `aria-expanded` on trigger button
- All icon-only buttons: `aria-label` required
- Focus ring: `focus-visible:ring-1 focus-visible:ring-ring` (shadcn default, already in globals)
- Minimum tap target: 44px on all interactive elements
