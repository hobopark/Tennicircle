| URL param -> Data query | /profile/[memberId] uses URL param to query another user's data |

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-3-05 | Information Disclosure | /profile/[memberId] | mitigate | RLS on player_profiles filters by community_id from JWT; server component checks viewer role is admin/coach before rendering contact info |
| T-3-08 | Spoofing | progress note form | mitigate | Server action validates auth and role before INSERT; RLS enforces coach/admin-only writes |
| T-3-09 | Tampering | memberId URL param | accept | memberId is a UUID; RLS prevents cross-community reads; worst case is coach sees another community member's profile which RLS blocks |
</threat_model>
