Build a custom CSS Grid calendar per UI-SPEC (Claude's discretion chose CSS Grid over FullCalendar for simplicity and zero bundle cost).

**Grid structure:**
- CSS Grid with `grid-template-columns: 60px repeat(7, 1fr)` (label column + 7 day columns)
- Rows represent 30-minute increments from 06:00 to 22:00 = 32 rows
- `grid-template-rows: 40px repeat(32, 48px)` — 40px header + 48px per time slot

**State management:**
- `useState` for `currentWeekStart` (Date) — default to start of current week (Monday)
- `useMemo` to compute the 7 days of the week from `currentWeekStart`
- `useMemo` to filter and position sessions within the current week

**Column headers (sticky):**
- First cell: empty (time label column)
- 7 cells: day name abbreviated + date number (e.g. "Mon 7"), `bg-muted`, `text-[14px]`, `font-normal`, `text-muted-foreground`
- `position: sticky; top: 0; z-index: 10`

**Time labels (left column):**
- Each row in column 1: time string ("6:00", "6:30", ..., "10:00 PM"), `text-[12px]`, `text-muted-foreground`

**Session blocks:**
- For each session, compute `gridRow` from `scheduled_at` time: `row = 2 + (hour - 6) * 2 + (minutes >= 30 ? 1 : 0)`
- `gridRowEnd = gridRow + Math.ceil(duration_minutes / 30)`
- `gridColumn` = day index + 2 (1-indexed, column 1 is time labels)
- Style: `bg-primary text-primary-foreground rounded-lg text-[13px] p-1 overflow-hidden cursor-pointer`
- Content: session title (truncated), time, attendee count badge
- Cancelled sessions: `bg-muted text-muted-foreground line-through` on title

**Session block click:** `<Link href={/coach/sessions/${session.id}}>` wrapping each block

**Navigation:**
- Prev/Next week buttons above the grid: `ChevronLeft` and `ChevronRight` icons from lucide-react
- "Today" button to reset to current week

**Responsive (mobile < 1024px):**
- Switch to 3-column view: `grid-template-columns: 60px repeat(3, 1fr)` showing today ± 1 day

**Empty state (UI-SPEC):**

Replace existing content with:

2. Pass sessions to `WeekCalendarGrid`
3. Layout: page heading "Schedule" (28px display), "Create session" link button (top right), then the calendar grid

```typescript
import { createClient } from '@/lib/supabase/server'
import { WeekCalendarGrid } from '@/components/calendar/WeekCalendarGrid'
import Link from 'next/link'
import { Plus } from 'lucide-react'

export default async function CoachPage() {
  const supabase = await createClient()
  const { data: { user } } = await supabase.auth.getUser()

  // Get member ID for this coach
  const { data: member } = await supabase
    .from('community_members')
    .select('id')
    .eq('user_id', user!.id)
    .single()

  // Two-query strategy: template-owned sessions + co-coached sessions (merged in JS)
  // Query 1: Sessions from templates this coach owns
  const { data: ownedSessions } = await supabase
    .from('sessions')
    .select('*, session_rsvps(count)')
    .in('template_id', (
      await supabase
        .from('session_templates')
        .select('id')
        .eq('coach_id', member!.id)
    ).data?.map(t => t.id) ?? [])
    .order('scheduled_at', { ascending: true })

  // Query 2: Sessions where this coach is a co-coach via session_coaches
  const { data: coCoachSessionIds } = await supabase
    .from('session_coaches')
    .select('session_id')
    .eq('member_id', member!.id)

  const coCoachIds = coCoachSessionIds?.map(sc => sc.session_id) ?? []
  const { data: coCoachedSessions } = coCoachIds.length > 0
    ? await supabase
        .from('sessions')
        .select('*, session_rsvps(count)')
        .in('id', coCoachIds)
        .order('scheduled_at', { ascending: true })
    : { data: [] }

  // Merge and deduplicate by session id
  const sessionMap = new Map<string, any>()
  for (const s of [...(ownedSessions ?? []), ...(coCoachedSessions ?? [])]) {
    if (!sessionMap.has(s.id)) sessionMap.set(s.id, s)
  }
  const sessions = Array.from(sessionMap.values()).sort(
    (a, b) => new Date(a.scheduled_at).getTime() - new Date(b.scheduled_at).getTime()
  )

  return (
    <div className="min-h-screen bg-background">
      <div className="max-w-6xl mx-auto px-4 py-8">
        <div className="flex items-center justify-between mb-6">
          <h1 className="font-display text-[28px] font-bold text-foreground">Schedule</h1>
          <Link href="/coach/sessions/new" className="inline-flex items-center gap-2 bg-primary text-primary-foreground px-4 py-2 rounded-lg text-sm font-medium hover:bg-primary/90">
            <Plus className="w-4 h-4" /> Create session
          </Link>
        </div>
        <WeekCalendarGrid sessions={sessions ?? []} />
      </div>
    </div>
  )
}
```

Note: Two separate queries are used because the Supabase JS SDK does not support nested `.or()` sub-selects. Results are merged and deduplicated in application code. This is the concrete strategy — no implementation-time decision needed.
