## Spacing Scale

All values are multiples of 4. Source: 8-point scale default — confirmed by Phase 1 code patterns (p-6 = 24px, p-8 = 32px, gap-4 = 16px observed in AuthPage/WelcomePage).

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gaps, avatar overlap offset |
| sm | 8px | Inline field spacing, chip padding |
| md | 16px | Default element spacing, card inner padding (mobile) |
| lg | 24px | Card inner padding (desktop), section gaps |
| xl | 32px | Card inner padding on sm: breakpoint, layout column gaps |
| 2xl | 48px | Major section breaks, page top padding |
| 3xl | 64px | Page-level vertical rhythm (not used in card views) |

Exceptions:
- Touch targets for RSVP and cancel buttons: minimum 44px height (mobile accessibility)
- Coach calendar time-slot row height: 48px per 30-minute increment
- Avatar preview stack: avatars sized 28px, overlapped by -8px (negative xs)

## Component Patterns

### Session Card (Client view — D-07, D-09)

Layout: full-width card, `bg-card` background, `rounded-2xl`, `border border-border`, `p-4` (mobile) / `p-6` (sm:).

Content order (top to bottom):
1. Date + time row: `CalendarDays` icon (14px, muted-foreground) + date string, `Clock` icon + time — 14px label weight 400
2. Venue row: `MapPin` icon + venue name — 14px label weight 400
3. Coach name row: coach display name — 14px label, muted-foreground
4. Spots remaining: `Users` icon + "{N} spots left" — 14px label; accent color (`#C4D82E`) when ≤ 3, muted-foreground otherwise
5. Attendee avatar preview: up to 4 avatars, 28px circles, overlapping by -8px, initials fallback if no avatar; "+N more" label in 14px muted-foreground if overflow
6. Action button row: right-aligned, full-width on mobile

**Cancelled session card variant:** Same layout with `opacity-60`, strikethrough on date/time row, "Cancelled: {reason}" in 14px destructive color replacing action button.

### RSVP Button States

| State | Label | Style |
|-------|-------|-------|
| Available | "Join session" | `variant="default"` (primary fill), h-11 |
| Full — not waitlisted | "Join waitlist" | `variant="outline"` with secondary (coral) border and text, h-11 |
| Waitlisted | "3rd on waitlist" (position inline) | `variant="outline"` secondary (coral), disabled-looking but tappable to cancel |
| Confirmed | "You're in" | `variant="ghost"` success green text + `CheckCircle` icon, h-11 — tappable to cancel |
| Loading | Spinner inside button, label hidden | width preserved, no layout shift |

### RSVP Confirmation Dialog (D-10)

Trigger: tap "Join session" button.
Component: `@base-ui/react` Dialog.
Content:
- Heading (20px, 700): "Join this session?"
- Body (16px, 400): "{Day} at {time} · {Venue}" — specific session details
- Buttons: "Confirm" (primary fill, right) + "Cancel" (ghost, left)
- Dialog max-width: 400px, centered, `bg-popover`, `rounded-2xl`

### Cancel RSVP Dialog (D-12)

Trigger: tap confirmed RSVP state or "Cancel" from session detail.
Content:
- Heading (20px, 700): "Cancel your spot?"
- Body (16px, 400, muted-foreground): "Heads up — if you're a regular, let your coach know you won't be there."
- Buttons: "Yes, cancel" (destructive fill, right) + "Keep my spot" (ghost, left)

### Edit Scope Dialog (D-14)

Trigger: coach taps "Edit" on a recurring session instance.
Content:
- Heading (20px, 700): "Edit this session?"
- Two choices as radio-style option cards (not radio inputs — full-card tap targets):
  - "This session only" — body 16px description: "Only {date} will change"
  - "This and all future sessions" — body 16px description: "All sessions from {date} onward will change"
- "Continue" button (primary fill) below choices

### Session Cancel Dialog (Coach — D-17)

Trigger: coach taps "Cancel session" on session detail.
Content:
- Heading (20px, 700): "Cancel this session?"
- Required text input: "Reason (required)" — placeholder "e.g. Rain, Public holiday"
- Validation: cannot submit with empty reason; inline error below field: "Please enter a reason for attendees"
- Buttons: "Cancel session" (destructive fill, right) + "Keep session" (ghost, left)

### Coach Weekly Calendar Grid (D-06)

Layout: CSS Grid — 7 columns (Mon–Sun) + 1 label column for time, rows = 30-minute increments from 06:00 to 22:00 (32 rows). Session blocks span rows proportional to duration.

- Grid background: `bg-background`
- Column headers: day name + date in 14px label, `bg-muted` row
- Cell background: `bg-card` for current week cells
- Session block: `bg-primary` fill, `text-primary-foreground`, `rounded-lg`, `text-[13px]`, shows title + time + attendee count
- Cancelled session block: `bg-muted` fill, `text-muted-foreground`, strikethrough title
- Scroll: vertical scroll within grid container; column headers stick at top (sticky)
- Mobile: collapses to 3-column view (today ± 1 day) with horizontal swipe navigation

### Session Detail Panel (Coach — D-15)

Layout: full page (not drawer) at `/coach/sessions/[sessionId]`.

Sections (top to bottom):
1. Session header: date, time, venue, coach names — 20px heading, below 14px metadata
2. Edit button: top-right, `Pencil` icon + "Edit" label, `variant="ghost"`
3. Cancel session button: top-right secondary action, destructive ghost, `XCircle` icon
4. Confirmed attendees section heading (16px, 700) + list of names with promote/remove actions (coach only sees this for waitlisted)
5. Waitlist section heading (16px, 700) + ordered list; each row: position badge (coral, 14px) + name + "Promote" button (primary outline, sm) + "Remove" button (destructive ghost, sm)
6. Empty state for each section if no attendees (see Copywriting below)

### Session Creation Form (D-01, D-03, D-04)

Layout: single page at `/coach/sessions/new`, max-width 560px centered.
Fields (top to bottom):
1. Title — text input, required
2. Day of week — select/dropdown (Mon–Sun), required
3. Start time — time picker (native `<input type="time">`), required
4. Duration — select (30min, 45min, 60min, 90min, 120min), default 60min
5. Venue — text input with autocomplete popover (previously-used venues), required
6. Court number — text input, optional, placeholder "Assign on the day"
7. Capacity — number input, min 1, required
8. Start date — date picker (native `<input type="date">`), required
9. End date — date picker, optional (leave blank for open-ended)
10. Co-coaches — multi-select from community coach list (D-16), optional

Submit: "Create sessions" button (primary fill, full width, h-11)

Form validation: inline field errors below each field (14px, destructive color), shown on blur + on submit attempt. Pattern matches Phase 1 Zod 4 + useActionState.

### Skeleton / Loading States

Session card skeleton: same card dimensions, `bg-muted` blocks for each text row, `animate-pulse`, `rounded`.
Calendar skeleton: full grid with `bg-muted` column headers and `animate-pulse` cell fills.
Session detail skeleton: header block + two list skeletons.

| Primary CTA — client RSVP | "Join session" | Verb + noun — from D-10 |
| Primary CTA — coach create | "Create sessions" | Plural — one form creates multiple instances |
| Waitlist CTA | "Join waitlist" | From D-11 |
| Confirmed state label | "You're in" | Reassuring, concise |

| Client empty state — body | "Your coach hasn't scheduled any sessions yet. Check back soon." | Blame-free, no action required from client |
| Coach empty state — heading | "Your schedule is clear" | Positive framing |
| Coach empty state — body | "Create a recurring session to get started." | Direct next step |
| Coach empty state — CTA | "Create session" | Button below empty state body |
| Confirmed attendees empty | "No confirmed attendees yet" | Inside session detail panel |
| Waitlist empty | "No one on the waitlist" | Inside session detail panel |
| RSVP dialog heading | "Join this session?" | From D-10 |
| Cancel RSVP dialog heading | "Cancel your spot?" | Possessive — user owns their spot |
| Cancel RSVP reminder | "Heads up — if you're a regular, let your coach know you won't be there." | Informational, not blocking (D-12) |
| Cancel RSVP confirm | "Yes, cancel" | Explicit |
| Cancel RSVP dismiss | "Keep my spot" | Positive cancel label |
| Edit scope dialog heading | "Edit this session?" | Direct |
| Edit scope — this only | "This session only" | Body: "Only {date} will change" |
| Edit scope — future | "This and all future sessions" | Body: "All sessions from {date} onward will change" |
| Session cancel dialog heading | "Cancel this session?" | Coach action |
| Session cancel reason placeholder | "e.g. Rain, Public holiday" | Guides input |
| Session cancel reason error | "Please enter a reason for attendees" | Tells coach why it's required |
| Session cancel confirm | "Cancel session" | Matches action exactly |
| Session cancel dismiss | "Keep session" | |

## States to Implement

All views must implement these states explicitly:

| View | States Required |
|------|----------------|
| Client sessions list | Loading (skeleton), Empty, Populated, Error |
| Session card | Available, Confirmed, Waitlisted, Full-but-not-joined, Cancelled |
| RSVP button | Idle, Loading (pending), Confirmed, Waitlisted, Error |
| Coach calendar | Loading (skeleton), Empty, Populated (with sessions) |
| Coach session detail | Loading (skeleton), With attendees, No attendees, With waitlist, No waitlist |
| Session creation form | Idle, Submitting, Field errors, Server error |
| RSVP dialog | Open/closed (no intermediate states) |
| Cancel dialog | Open/closed |
| Edit scope dialog | Choice unselected (Continue disabled), Choice selected (Continue enabled) |
| Session cancel dialog | Idle, Reason field empty (submit disabled), Submitting |

## Responsive Breakpoints

| Breakpoint | Behavior |
|------------|----------|
| Default (mobile) | Single-column card list; full-width buttons; bottom-anchored dialogs |
| sm (640px+) | Cards at max-width 560px centered; dialogs centered with max-width 400px; card padding increases to p-6 |
| lg (1024px+) | Coach calendar shows 7-day grid; client view unchanged (card-based stays card-based) |

Mobile-specific:
- RSVP dialogs: slide up from bottom (sheet pattern) on screens < 640px; centered modal on sm+
- Coach calendar: 3-day column view with prev/next navigation on screens < 1024px
