**Step 2: Create `vitest.config.ts`** in the project root:

```typescript
import { defineConfig } from 'vitest/config'
import path from 'path'

export default defineConfig({
  test: {
    environment: 'jsdom',
    globals: true,
    setupFiles: [],
    include: ['src/**/*.test.{ts,tsx}'],
    exclude: ['node_modules', '.next'],
  },
  resolve: {
    alias: {
      '@': path.resolve(__dirname, './src'),
    },
  },
})
```

The `@` alias must match `tsconfig.json` paths (`@/*` -> `./src/*`). The `jsdom` environment is needed for any component tests. Server action tests can override with `// @vitest-environment node` per file if needed.
