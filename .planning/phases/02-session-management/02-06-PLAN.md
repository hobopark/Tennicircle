**Step 1: Install required dependencies**

```bash
npm install date-fns
```

Note: react-hook-form and @hookform/resolvers were listed as optional in RESEARCH.md. If CreateSessionForm in Plan 03 used useActionState (Phase 1 pattern) instead of react-hook-form, these are not needed. Only install if Plan 03 summary indicates they were used. If not used, skip.

Verify installation:
```bash
node -e "require('date-fns')" && echo "date-fns OK"
```

**Step 2: Push database schema to Supabase**

```bash
npx supabase db push
```

If this prompts for confirmation interactively, use:
```bash
npx supabase db push --include-all
```

Or if SUPABASE_ACCESS_TOKEN is set:
```bash
SUPABASE_ACCESS_TOKEN=$SUPABASE_ACCESS_TOKEN npx supabase db push
```

If pg_cron is not available and the migration fails on the `cron.schedule` call, the migration should have an exception handler wrapping it. If it doesn't, temporarily comment out the pg_cron section, push, then re-add it.

**Step 3: Verify schema was applied**

```bash
npx supabase db diff
```

Should show no pending changes (all applied).

**Step 4: Run type check**

```bash
npx tsc --noEmit
```

Must pass with zero errors.

**Step 5: Run existing tests**

```bash
npx vitest run --reporter=verbose
```

Must pass (or report no test files if none exist yet for Phase 2).

  <what-built>
Complete session management system:
- Database schema with 4 tables, RLS, capacity trigger, session generation function
- Server actions for RSVP (join/cancel/promote) and session management (create/edit/cancel)
- Coach weekly calendar grid at /coach
- Coach session detail page with attendee/waitlist management at /coach/sessions/[id]
- Session creation form at /coach/sessions/new
- Session editing with this/future scope at /coach/sessions/[id]/edit
- Client sessions page with card-based layout at /sessions
- RSVP confirmation and cancel dialogs
  </what-built>
  <how-to-verify>
**Prerequisites:** Make sure you're logged in as a coach user and have the dev server running (`npm run dev`).

**Coach flow:**
1. Visit http://localhost:3000/coach — verify weekly calendar grid renders (should show empty state "Your schedule is clear" if no sessions)
2. Click "Create session" — verify form renders at /coach/sessions/new with all fields (title, day, time, duration, venue, capacity, dates, co-coaches)
3. Fill in: Title "Tuesday Drills", Day "Tuesday", Time "18:00", Duration "60 min", Venue "Moore Park", Capacity "4", Start date (today), End date (8 weeks out)
4. Click "Create sessions" — verify redirect to /coach and sessions appear in the calendar grid
5. Click a session block — verify session detail page loads with "No confirmed attendees yet" and "No one on the waitlist"
6. Click "Edit" — verify edit scope dialog appears with "This session only" / "This and all future sessions"
7. Click "Cancel session" on the detail page — verify dialog requires a reason before submitting

**Client flow (log in as a client user assigned to the above coach):**
8. Visit http://localhost:3000/sessions — verify session cards appear showing date, time, venue, coach name, spots remaining
9. Click "Join session" on a card — verify confirmation dialog appears with session details
10. Click "Confirm" — verify button changes to "You're in" (green with checkmark)
11. Click "You're in" — verify cancel dialog appears with courtesy message
12. Click "Yes, cancel" — verify button returns to "Join session"

**Capacity + Waitlist:**
13. RSVP enough users to fill a session (4 RSVPs for capacity 4)
14. As another client, verify button shows "Join waitlist" instead of "Join session"
15. Join waitlist — verify position shows (e.g. "1st on waitlist")
16. As coach, visit session detail — verify waitlist shows with Promote button
17. Click Promote — verify member moves to confirmed list

**Edge cases:**
18. Verify cancelled sessions show "Cancelled: {reason}" on both coach calendar and client cards
  </how-to-verify>
