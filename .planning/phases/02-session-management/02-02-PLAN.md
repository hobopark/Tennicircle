**Function 1: `rsvpSession(sessionId: string): Promise<RsvpActionResult>`**

Flow (from RESEARCH Pattern 4):
1. `const supabase = await createClient()` then `supabase.auth.getUser()` â€” return error if not authenticated
2. Get member record: `supabase.from('community_members').select('id').eq('user_id', user.id).single()`
3. Check session not cancelled: `supabase.from('sessions').select('capacity, cancelled_at').eq('id', sessionId).single()` â€” if `cancelled_at` is not null, return `{ success: false, error: 'This session has been cancelled' }`
4. Count confirmed RSVPs: `supabase.from('session_rsvps').select('*', { count: 'exact', head: true }).eq('session_id', sessionId).eq('rsvp_type', 'confirmed').is('cancelled_at', null)`
5. If `confirmedCount < capacity`: insert with `rsvp_type: 'confirmed'`, `waitlist_position: null`
6. If `confirmedCount >= capacity`: count current waitlisted (same query pattern but `eq('rsvp_type', 'waitlisted')`), insert with `rsvp_type: 'waitlisted'`, `waitlist_position: waitlistCount + 1`
7. Insert into session_rsvps: `supabase.from('session_rsvps').insert({ session_id: sessionId, member_id: member.data.id, community_id: user.app_metadata.community_id, rsvp_type, waitlist_position })`
8. If insert error contains 'capacity', return `{ success: false, error: 'This session is now full. You\'ve been added to the waitlist.' }` â€” this catches the DB trigger exception
9. `revalidatePath('/sessions')` and `revalidatePath('/coach')` â€” invalidate both views
10. Return `{ success: true, rsvpType, waitlistPosition }`

**Function 2: `cancelRsvp(sessionId: string): Promise<SessionActionResult>`**

Flow:
1. Auth check (same pattern as above)
2. Get member record
3. Update `session_rsvps` SET `cancelled_at = new Date().toISOString()` WHERE `session_id = sessionId` AND `member_id = member.id` AND `cancelled_at IS NULL`
4. If the cancelled RSVP was `waitlisted`, resequence remaining waitlist positions for the session:
   - Select all non-cancelled waitlisted RSVPs for the session, ordered by `created_at`
   - For each, update `waitlist_position` to its 1-based index in the result set
5. `revalidatePath('/sessions')` and `revalidatePath('/coach')`
6. Return `{ success: true }`

**Function 3: `promoteFromWaitlist(rsvpId: string): Promise<SessionActionResult>`**

Flow:
1. Auth check â€” user must be coach or admin (`user.app_metadata.user_role` is `'coach'` or `'admin'`). Return `{ success: false, error: 'Only coaches can promote from waitlist' }` if not.
2. Get the RSVP record: `supabase.from('session_rsvps').select('*, sessions(capacity)').eq('id', rsvpId).single()`
3. Verify RSVP is currently `waitlisted` and not cancelled
4. Count confirmed RSVPs for that session (non-cancelled)
5. If confirmed count >= session capacity, return `{ success: false, error: 'Session is at capacity. Cancel an existing RSVP first.' }`
6. Update RSVP: `supabase.from('session_rsvps').update({ rsvp_type: 'confirmed', waitlist_position: null }).eq('id', rsvpId)`
7. Resequence remaining waitlist positions for the session (same logic as cancelRsvp step 4)
8. `revalidatePath('/sessions')` and `revalidatePath('/coach')`
9. Return `{ success: true }`

**Function 4: `removeFromWaitlist(rsvpId: string): Promise<SessionActionResult>`**

Flow (coach-scoped removal of a waitlisted member):
1. Auth check â€” user must be coach or admin (`user.app_metadata.user_role` is `'coach'` or `'admin'`). Return `{ success: false, error: 'Only coaches can remove from waitlist' }` if not.
2. Get the RSVP record: `supabase.from('session_rsvps').select('*').eq('id', rsvpId).single()`
3. Verify RSVP is currently `waitlisted` and not cancelled. Return error if not.
4. Update RSVP: `supabase.from('session_rsvps').update({ cancelled_at: new Date().toISOString() }).eq('id', rsvpId)`
5. Resequence remaining waitlist positions for the session (same logic as cancelRsvp step 4)
6. `revalidatePath('/sessions')` and `revalidatePath('/coach')`
7. Return `{ success: true }`

Important: Use `getUser()` never `getSession()` (Phase 1 decision). Always check `cancelled_at is null` when counting RSVPs.
