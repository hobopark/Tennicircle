## Spacing Scale

Standard 8-point scale. All spacing uses Tailwind utilities mapped to this scale.

| Token | Value | Tailwind Class | Usage |
|-------|-------|----------------|-------|
| xs | 4px | `gap-1`, `p-1` | Icon gaps, badge padding, inline micro-spacing |
| sm | 8px | `gap-2`, `p-2` | Button icon gap, avatar gap in attendance preview |
| md | 16px | `px-4`, `gap-4` | Default card padding, nav horizontal padding |
| lg | 24px | `mb-6`, `gap-6` | Section gaps, heading margin-bottom |
| xl | 32px | `py-8`, `gap-8` | Page vertical padding |
| 2xl | 48px | `py-12` | Empty state vertical centering padding |
| 3xl | 64px | `py-16` | Full-page empty state vertical padding |

Exceptions:
- Day/week toggle button group: 2px internal gap between segments (design system internal)
- Touch targets on mobile: minimum 44px height for all tappable row items (roster rows, event card RSVP button)
- Attendance avatar stack: -8px negative margin overlap between stacked avatars (`-ml-2`)

## Color

Source: `src/app/globals.css` CSS custom properties.

| Role | CSS Token | Light Mode Value | Usage |
|------|-----------|-----------------|-------|
| Dominant (60%) | `--background` | `hsl(60 10% 98%)` warm off-white | Page background, full-bleed areas |
| Secondary (30%) | `--card` / `--muted` | `hsl(0 0% 100%)` / `hsl(150 10% 94%)` | Cards, nav bar (`bg-popover`), sidebar, roster rows |
| Accent (10%) | `--primary` | `hsl(152 65% 42%)` tennis court green | See reserved list below |
| Destructive | `--destructive` | `hsl(0 84.2% 60.2%)` | Event cancellation, RSVP cancellation confirmation button only |

**Accent (`--primary` green) reserved for:**
- Primary CTA buttons ("Create Event", "RSVP", "Add Session")
- Active nav link indicator
- Day/week toggle selected segment
- Today's date highlight in calendar header
- Confirmed session blocks in calendar grid
- Focus ring (already set via `--ring`)

**Secondary yellow-orange (`--secondary`, `hsl(45 100% 58%)`) reserved for:**
- Announcement pinned card left border accent (`border-l-4 border-secondary`)
- "New" or "Pinned" badge on announcement cards

**Event type badges use semantic colours:**
- Tournament: `bg-secondary/20 text-secondary-foreground` (warm yellow-orange tint)
- Social Event: `bg-accent text-accent-foreground` (subtle green tint)
- Open Session: `bg-primary/15 text-primary` (primary green ghost)
- Announcement: `bg-secondary/30 text-secondary-foreground` with left border

**Do not use raw hex values in code.** Always reference CSS tokens via Tailwind utility classes.

## Component Inventory

### Coach Dashboard (`/coach`)

**Day/Week Toggle**
- Use `ToggleGroup` from shadcn with two segments: "Day" | "Week"
- Default: "Week" on desktop (≥768px), "Day" on mobile (<768px) — detect via `useMediaQuery` or CSS-only approach with hidden/shown variants
- Placement: inline with page title, right-aligned, same row as "Create session" button

**Day View** (new, D-01)
- Replaces the 7-column grid with a single-column time-block list
- Each time block: time label left, session card right
- Session card in day view shows: title, time range, venue, confirmed/capacity count, first 3 attendee avatars (stacked, 24px size), overflow count "+N"
- No horizontal scroll — single column fills full container width
- Time range displayed as "6:00 AM – 7:30 AM"

**Week View** (existing `WeekCalendarGrid` — extend, do not rewrite)
- Add day/week toggle prop; no other changes to grid rendering
- Inline attendance preview already present (attendeeCount / capacity) — keep

**Today's Sessions Card Stack** (D-03)
- Shown above the calendar in both day and week views when there are sessions today
- Each card: session title, time, venue, `confirmed / capacity` pill, avatar strip (first 3 faces + "+N"), "View" link
- Card uses `bg-card border border-border rounded-2xl p-4`

**Player Roster (`/coach/clients`)** (D-02)
- Table-style list, one row per player
- Columns: Avatar + Name, Skill Level (coach-assessed badge), Last Attended, First Lesson Date, row arrow link
- On mobile: stack Name + metadata vertically, remove Last Attended column
- Empty state: "No players yet. Invite clients to get started." + "Invite client" CTA

### Events Page (`/events`) (D-07, D-08)

**Top-level tabs: "Official" | "Community"**
- Use shadcn `Tabs` component (already installed)
- Default: "Official" tab active
- Tab switcher sits below page title, full-width on mobile

**Official Tab**
- Pinned announcements at top (if any): card with `border-l-4 border-secondary bg-secondary/10 rounded-2xl p-4`
- Event cards below announcements, sorted by date ascending
- "Post announcement" button visible to coach/admin only (right of section heading)

**Community Tab**
- Event cards sorted by date ascending
- "Create Event" button visible to all roles — top right of page, primary CTA

**Event Card** (D-10)
- Structure: type badge (top left) + title (heading size) + date/time + venue + "X spots remaining" or "Full" pill + RSVP button
- Container: `bg-card border border-border rounded-2xl p-4`
- RSVP button: `variant="default"` (primary green) when not RSVPed; `variant="outline"` with checkmark when RSVPed
- "Full" state: RSVP button disabled, pill shows `bg-destructive/10 text-destructive`
- Tappable: entire card is a link to `/events/[eventId]`, RSVP button uses `stopPropagation`

**Event Creation Flow** (D-04)
- Single "Create Event" button opens a `Dialog`
- Step 1 of dialog: type selector — 3 large radio-style cards: "Tournament", "Social Event", "Open Session"
  - Each option card: icon + label + one-line description
  - Selected state: `border-primary bg-accent` ring
- Step 2: form adapts to selected type. All types share: Title (required), Date & Time (required), Venue (required, reuse `VenueAutocomplete`), Capacity (optional)
  - Tournament only: Image upload field for draw (label: "Draw image (optional — upload bracket photo")
  - Social Event: Description text area
  - Open Session: Description text area
- Form submission: server action, `useActionState` pattern, inline field errors
- After success: dialog closes, toast "Event created", page revalidates

**Announcement Creation** (D-09)
- Coach/admin only
- "Post announcement" button opens same `Dialog` with type pre-set to "Announcement"
- Fields: Title (required, max 80 chars), Body (required, max 400 chars, textarea)
- Announcements do not have RSVP or capacity fields

### Client Dashboard (`/sessions`) (D-11, D-12, D-13)

**Personalized Greeting**
- `"Good morning, {first_name}"` — time-of-day greeting (morning/afternoon/evening)
- Sub-line: quick stats in one sentence: "X sessions this month · Y upcoming RSVPs · Member since {month year}"
- Stats use `text-muted-foreground text-sm`

**Upcoming Sessions Section**
- Section heading: "Your upcoming sessions" (20px bold)
- Shows next 3–5 sessions as horizontal-scroll card row on mobile, vertical stack on desktop ≥768px
- Each session card: title, date + time, venue, RSVP status badge (Confirmed / Waitlisted), RSVP/Cancel button
- "No upcoming sessions" empty state inside section (not full-page)

**Upcoming Events Section**
- Section heading: "Events in your community" (20px bold)
- Shows next 3–5 events (across both Official and Community tabs) as cards
- Each card: same event card pattern as Events page (compact version without full venue line)
- "Explore events" link below the list — navigates to `/events`

**Recent Announcements Section**
- Section heading: "Announcements" (20px bold)
- Shows last 2 announcements as compact cards
- "View all" link navigates to `/events` (Official tab)
- If no announcements: section is omitted entirely (do not show empty state)

## Page Layout Contract

All pages follow this shell (established in Phase 1–3):

```
<AppNav />                        — full-width, bg-popover, border-b
<div className="min-h-screen bg-background">
  <div className="max-w-6xl mx-auto px-4 py-8">
    {page content}
  </div>
</div>
```

`max-w-4xl` is used for narrower content pages (auth, profile). Use `max-w-6xl` for dashboard, events, and roster.

Mobile breakpoint: Tailwind `md` (768px). Below 768px:
- Nav links collapse to horizontal scroll (no hamburger menu in Phase 4 — deferred)
- Calendar defaults to Day view
- Card rows become vertical stacks
- Roster table collapses to name + last-attended only

## Interaction Contracts

### Day/Week Toggle
- Switching views is instant (no loading state) — data already loaded
- Selected view persists in `localStorage` key `tennis-schedule-view`; falls back to "week" on first visit desktop, "day" on mobile

### Event RSVP
- Optimistic UI: button flips to "Cancel RSVP" immediately on click
- On server error: revert to "RSVP" + show error toast
- Full event: button disabled, shows "Full" — no optimistic state
- Reuse `src/lib/actions/rsvps.ts` pattern exactly (join / cancel / promote)

### Event Creation Dialog
- Step 1 → Step 2 transition: slide-right animation within dialog (no separate page)
- Back button on Step 2 returns to Step 1 with type retained
- Form preserves values if user navigates back
- On submit: button shows loading spinner, disabled
- Success: dialog closes, Sonner toast `"Event created"` (richColors green), page data revalidates via `router.refresh()`

### Tournament Draw Upload
- Accept: `image/png`, `image/jpeg` only
- Max size: 5 MB — reject client-side with inline error before upload
- Upload to Supabase Storage bucket `tournament-draws`
- Show image preview after selection (before submit)
- On submit: upload first, then create event with `draw_image_url`

### Announcement Cards
- Pinned at top of Official tab — always above event cards
- Not tappable as a full card link; body text is fully visible inline
- Coach/admin sees "Edit" icon button (pencil, lucide `Pencil`) top-right of card
- Edit opens same dialog pre-filled

### Skeleton Loading States
- Coach dashboard: skeleton for "Today's sessions" card stack (3 placeholder cards)
- Events page: skeleton grid of 4 event card outlines on initial load
- Roster page: skeleton for 6 row outlines
- Use shadcn `Skeleton` component with `animate-pulse`
- Skeleton height matches real content height to avoid layout shift
