**Create `src/lib/actions/events.ts`** with `'use server'` directive. Follow the exact pattern from `rsvps.ts`: createClient -> getUser -> getJWTClaims -> member lookup -> DB operation -> revalidatePath.

**`createEvent` action:**
- Signature: `export async function createEvent(_prevState: EventActionResult, formData: FormData): Promise<EventActionResult>`
- Get user via `supabase.auth.getUser()`, get claims via `getJWTClaims(supabase)`, get member record
- Parse formData with `CreateEventSchema.safeParse(Object.fromEntries(formData))`
- On validation failure: return `{ success: false, fieldErrors: parsed.error.flatten().fieldErrors }`
- **CRITICAL (T-04-02, D-09):** Compute `is_official` SERVER-SIDE: `const isOfficial = claims.user_role === 'coach' || claims.user_role === 'admin'` — do NOT read is_official from formData
- Combine `starts_at_date` and `starts_at_time` into ISO string: `new Date(\`\${parsed.data.starts_at_date}T\${parsed.data.starts_at_time}\`).toISOString()`
- Convert empty string/zero `capacity` and `duration_minutes` to null before insert
- Insert into `events` table with all fields including `community_id`, `created_by: member.id`, `is_official: isOfficial`
- `revalidatePath('/events')` and `revalidatePath('/sessions')`
- Return `{ success: true, data: insertedEvent }`

**`rsvpEvent` action:**
- Signature: `export async function rsvpEvent(eventId: string): Promise<EventRsvpActionResult>`
- Auth check, claims check, member lookup (same pattern)
- Fetch event to check capacity and cancelled_at
- If event has capacity: count confirmed RSVPs where `cancelled_at is null`; if count >= capacity, set `rsvpType = 'waitlisted'`; else `rsvpType = 'confirmed'`
- For waitlisted: compute `waitlist_position` by counting existing waitlisted RSVPs + 1
- Insert into `event_rsvps` with `event_id`, `member_id: member.id`, `community_id`, `rsvp_type`, `waitlist_position` (null if confirmed)
- Handle unique constraint error (already RSVP'd) — return `{ success: false, error: 'You have already RSVP\'d to this event' }`
- `revalidatePath('/events')` and `revalidatePath('/sessions')`
- Return `{ success: true, rsvpType }`

**`cancelEventRsvp` action:**
- Signature: `export async function cancelEventRsvp(eventId: string): Promise<EventRsvpActionResult>`
- Auth + member lookup
- Update `event_rsvps` set `cancelled_at = new Date().toISOString()` where `event_id` and `member_id` match and `cancelled_at is null`
- `revalidatePath('/events')` and `revalidatePath('/sessions')`
- Return `{ success: true }`

**`deleteEvent` action:**
- Signature: `export async function deleteEvent(eventId: string): Promise<EventActionResult>`
- Auth + member + claims
- Verify user is creator or admin: fetch event, check `event.created_by === member.id || claims.user_role === 'admin'`
- Delete from `events` where `id = eventId` (cascade deletes RSVPs)
- `revalidatePath('/events')`
- Return `{ success: true }`

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Client form -> createEvent action | formData is untrusted; is_official must be server-derived |
| Client -> rsvpEvent action | eventId comes from client; capacity check runs server-side |
| Client -> DrawImageUpload | File upload content could be malicious |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-02 | Tampering | createEvent is_official | mitigate | is_official derived from `claims.user_role` in server action, NOT from formData |
| T-04-05 | Elevation | createAnnouncement | mitigate | Role check `claims.user_role in ('coach', 'admin')` before INSERT |
| T-04-06 | Spoofing | rsvpEvent | mitigate | member_id derived from auth.uid() lookup, not from client input |
| T-04-07 | Denial of Service | DrawImageUpload | mitigate | Client-side 5MB limit + type validation; Supabase Storage has server-side limits |
| T-04-08 | Tampering | deleteEvent | mitigate | Server action verifies user is event creator or admin before deletion |
</threat_model>
