# Phase 1 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Scope:** /auth page (login + signup tabs), email verification pending page, welcome page (post-login), and role-based redirect logic. No session or dashboard UI in this phase.

Follow this existing pattern — do not add a `tailwind.config.ts` unless the executor encounters a hard blocker.

## Spacing Scale

Declared values (multiples of 4 only):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline chip padding |
| sm | 8px | Compact element spacing, icon-to-label gaps |
| md | 16px | Default element spacing, screen edge padding |
| lg | 24px | Card internal padding, section gaps |
| xl | 32px | Form section breaks, layout column gaps |
| 2xl | 48px | Major section breaks, page top padding |
| 3xl | 64px | Full-page vertical centering offset |

Exceptions:
- Touch targets (buttons, tab items): minimum 52px height (accessibility)
- Input fields: exactly 48px height
- Screen edge padding: 16px on mobile, 24px on desktop (md/lg tokens)

## Typography

**Font families:**
- Display/Headings: `Nunito`, weight 700 (Bold)
- Body/UI: `Nunito Sans`, weight 400 (Regular)

**Declared weights: 400 (Regular) and 700 (Bold) only. No other weights are permitted.**

| Role | Size | Weight | Line Height | Font Family |
|------|------|--------|-------------|-------------|
| Body | 16px | 400 | 1.5 | Nunito Sans |
| Label / UI text | 14px | 400 | 1.4 | Nunito Sans |
| Heading (card/section) | 20px | 700 | 1.3 | Nunito |
| Display (page title) | 28px | 700 | 1.2 | Nunito |

**Usage notes:**
- Body (16px/400) — form field values, helper text, paragraph copy
- Label (14px/400) — form field labels, tab labels, button text, nav items, helper/legal text below buttons, skip links
- Heading (20px/700) — card titles, section headings, auth form heading ("Welcome back"), wordmark/logotype
- Display (28px/700) — page-level title on welcome page ("You're in, [name]")
- No italic variants in Phase 1
- No additional sizes. If something doesn't fit these 4 roles, reconsider the layout.

## Color

### Palette

All shadcn CSS variables are overridden in `globals.css` inside `:root`. Dark mode is NOT implemented in Phase 1 — the `@media (prefers-color-scheme: dark)` block should be removed or neutered. Replace the existing `--background: #ffffff` and `--foreground: #171717` defaults.

| Role | Hex | CSS Variable | Usage |
|------|-----|-------------|-------|
| Dominant (60%) | `#FAF6F1` | `--background` | Screen background, page surface |
| Secondary (30%) | `#F0EBE3` | `--card` / `--input` | Card surfaces, input field fill, tab bar |
| Card surface | `#FFFFFF` | `--popover` | White card on top of cream bg (auth card) |
| Primary text | `#1C1C1C` | `--foreground` | Body copy, headings, labels |
| Secondary text | `#8C8279` | `--muted-foreground` | Placeholder text, helper text, inactive tab label |
| Accent — Court Blue | `#2D5F8A` | `--primary` | Primary CTA button fill, active tab indicator, focus ring |
| Accent foreground | `#FFFFFF` | `--primary-foreground` | Text on primary button |
| Highlight — Tennis Yellow | `#C4D82E` | `--accent` | Success badges, active indicators, subtle highlights |
| Highlight foreground | `#1C1C1C` | `--accent-foreground` | Text on yellow highlight (dark for contrast) |
| Secondary action | `#C17C5E` | `--secondary` | Secondary/ghost button hover, badge highlight (deferred to later phases) |
| Border / divider | `#E8E2DA` | `--border` | Input borders, card dividers, tab underline |
| Destructive | `#C75D5D` | `--destructive` | Error messages, destructive button (not in Phase 1 UI but declared for consistency) |
| Ring (focus) | `#2D5F8A` | `--ring` | Focus outline, matches accent |

**Semantic status colors (declare in globals.css, not shadcn variables):**
- Success: `#5B9A6B`
- Error: `#C75D5D`
- Warning: `#C4944A`
- Info: `#2D5F8A` (matches Court Blue — replaces the separate info blue)

### Color contract rules (anti-patterns — enforce strictly)

- Do NOT use `#FFFFFF` as screen background — use `#FAF6F1`
- Do NOT use `#000000` for any text — use `#1C1C1C`
- Do NOT add visible borders to auth card — use shadow only
- Do NOT apply the accent color to non-interactive elements (accent is reserved — see below)
- Do NOT use Tennis Yellow for large surfaces — it's a highlight, not a fill

**Accent (`#2D5F8A` Court Blue) reserved for:**
1. Primary button fill and hover state
2. Active tab indicator (the underline/highlight on the selected tab)
3. Input field focus ring/border
4. Checkbox/radio checked state
5. Loading spinner (Phase 1 skeleton states)

**Highlight (`#C4D82E` Tennis Yellow) reserved for:**
1. Success/completion badges and checkmarks (later phases)
2. Active state dot indicators
3. Subtle highlight accents (e.g., welcome page icon glow)
4. NOT used for buttons, text, or large fills — highlight only

Accent is NOT used for: text links (use `#1C1C1C` underlined), passive icons, card backgrounds, or page sections.

**Rationale — Australian Open identity:** Court Blue (`#2D5F8A`) evokes the Australian Open hard court — instantly recognizable as tennis and locally relevant for a Sydney-based community. Tennis Yellow (`#C4D82E`) is a muted nod to the tennis ball — used sparingly as a highlight to avoid visual noise. Together they create a distinct tennis identity while keeping the warm, calm cream-base aesthetic from the design brief.

## Component Specifications

### Auth Card (the white card on /auth)

- Background: `#FFFFFF`
- Border radius: 16px
- Shadow: `0 2px 12px rgba(0, 0, 0, 0.08)` — no visible border
- Padding: 32px (xl token) on desktop, 24px (lg token) on mobile
- Max width: 440px, centered on page
- The card sits on the cream background (`#FAF6F1`)

### Tab Switcher (Login / Sign Up)

- Container background: `#F0EBE3` (warm sand pill)
- Border radius: 12px (pill container)
- Active tab: white background pill, `#1C1C1C` text, no accent fill on tab itself
- Active tab indicator: 2px `#2D5F8A` underline below the active tab label (inside the pill is also acceptable)
- Inactive tab: transparent, `#8C8279` text
- Tab height: 44px minimum
- Use shadcn `Tabs` component (`npx shadcn@latest add tabs`)

### Input Fields

- Background: `#F0EBE3` (warm sand)
- Border: 1px solid `#E8E2DA` default, 1.5px solid `#2D5F8A` on focus
- Border radius: 12px
- Height: 48px
- Font: Nunito Sans 16px weight 400
- Placeholder: Nunito Sans 16px weight 400, color `#8C8279`
- Padding: 0 16px
- Error state: border color `#C75D5D`, error message below field in `#C75D5D` at 14px/400
- Use shadcn `Input` component (`npx shadcn@latest add input`)

### Primary Button

- Background: `#2D5F8A` (Court Blue)
- Text: `#FFFFFF`, Nunito Sans 14px weight 400
- Border radius: 12px
- Height: 52px
- Width: 100% (full width within form context)
- Hover: `background-color` darken 8% → `#265178`
- Active/pressed: darken 12% → `#1F4466`
- Disabled: opacity 0.4, cursor not-allowed
- Loading state: spinner replaces button text (do not expand button width)
- Use shadcn `Button` component (`npx shadcn@latest add button`)

### Form Labels

- Font: Nunito Sans 14px weight 400, color `#1C1C1C`
- Spacing below label: 8px before input

### Loading Spinner (auth state check)

- Color: `#2D5F8A`
- Size: 24px
- Full-page centered on cream background while Supabase session check completes
- Do NOT show any nav or layout shell during loading

### Toast / Feedback (email verification sent)

- Background: `rgba(28, 28, 28, 0.9)` — charcoal 90% opacity
- Text: `#FFFFFF`, Nunito Sans 14px weight 400
- Border radius: 12px
- Padding: 12px 16px
- Position: fixed, bottom 24px, centered horizontally
- Animation: slide up from bottom (transform translateY)
- Use shadcn `Sonner` component for toasts (`npx shadcn@latest add sonner`)

### Welcome Page Card

- Same card spec as Auth Card (white, 16px radius, soft shadow)
- Icon: Tennis racket (lucide `CircleDot` or closest available — if no racket icon, use a tennis-adjacent icon like `Zap` or `Trophy`)

## Screen Designs

### /auth — Auth Page

**Layout:** Full-page cream background. Auth card centered vertically at 45% viewport height (slightly above center — more natural than dead center). TenniCircle wordmark above card. Tabs inside card.

**Primary focal point:** the auth card — all surrounding elements support it without competing.

**Page structure (top to bottom):**
1. TenniCircle logo/wordmark — Nunito Bold 20px (heading size), color `#1C1C1C`, centered, 48px above card
2. Auth card (440px max-width, white)
   - Heading: "Welcome back" (Login tab) / "Join TenniCircle" (Sign Up tab) — Nunito Bold 20px
   - Tab switcher: Login | Sign Up (inside card, below heading)
   - Form fields (see below)
   - Primary CTA button
   - Helper text below button

**Login tab fields:**
- Email (label: "Email address", placeholder: "you@example.com")
- Password (label: "Password", placeholder: "Your password")
- CTA button: "Log in"
- Below button: "Forgot password?" — Nunito Sans 14px, `#8C8279`, right-aligned (deferred functionality, link visible but unimplemented in Phase 1 — use `aria-disabled`)

**Sign Up tab fields:**
- Email (label: "Email address", placeholder: "you@example.com")
- Password (label: "Create a password", placeholder: "At least 8 characters")
- CTA button: "Create account"
- Below button: Helper text — "By signing up, you agree to our terms." — Nunito Sans 14px `#8C8279` (terms page not built — static text only)

**Inline error placement (D-03):**
- Error text appears 4px below the specific field that failed
- Color `#C75D5D`, font Nunito Sans 14px weight 400
- Field border turns `#C75D5D`
- Example: "Please enter a valid email address" / "Password must be at least 8 characters"

**Password requirements:** Minimum 8 characters. No complexity requirements. Error shows on submit (not on blur) — Claude's discretion resolved: validate on submit, show inline, remove error when user corrects the field.

### /auth — Email Verification Pending State

After successful sign-up, replace the form content (inside the same card) with:

- Heading: "Check your email" — Nunito Bold 20px
- Body: "We sent a confirmation link to **[email]**. Click it to activate your account." — 16px/400 with email in bold
- Helper: "Didn't get it? Check your spam folder or — " + "resend the email" link (Nunito Sans 14px, `#2D5F8A`, calls `resend()` once, then disables for 60s with countdown)
- Do NOT navigate away from /auth — update in-place

### /welcome — Welcome Page (D-11, post-login Phase 1 landing)

**Shown to:** All roles in Phase 1 (since role-specific dashboards don't exist yet)

**Layout:** Full-page cream background. Single card centered.

**Card content:**
- Icon: Lucide `Trophy` or `CircleDot`, size 40px, color `#2D5F8A`, centered at top of card
- Heading: "You're in!" — Nunito Bold 28px (display size)
- Subheading: "Set up your profile to get the most out of TenniCircle." — Nunito Sans 16px `#8C8279`
- CTA button: "Set up my profile" (full-width, primary style)
- Skip link: "I'll do this later" — Nunito Sans 14px `#8C8279`, centered, below button, 16px margin top

**Role routing note:** Once the user has an established role AND the role-specific page exists (Phase 2+), the welcome page is bypassed. In Phase 1, all post-login traffic lands here.

### Loading State (auth check)

- Full cream page background
- 24px Court Blue spinner, centered horizontally and vertically
- No layout shell, no nav, no card
- Shown for maximum 2 seconds; if Supabase call takes longer, show spinner until resolved

## Interaction Contract

### Form validation timing

- Validate on submit (not on blur)
- On submit with errors: scroll to first error, focus the first errored field
- Clear field-level error when user begins typing in that field (not on blur)
- Server errors (wrong credentials, account not found): show as a general error above the submit button — not tied to a specific field — in a light red banner (`#FFF0F0` bg, `#C75D5D` border, `#C75D5D` text)

### Tab switching

- Switching Login/Sign Up tabs clears all field values and all error states
- No animation between tabs (simple show/hide via shadcn Tabs)

### Button loading state

- On submit: button text replaced with 16px spinner (white), button disabled, width unchanged
- If request fails: spinner removed, error displayed, button re-enabled
- Never show two buttons simultaneously

### Mobile behavior

- Stack form vertically, full-width inputs
- Auth card: remove card styling on screens < 440px (no shadow, no border-radius, full-width)
- Screen edge padding: 16px on mobile (md token)
- Minimum tap targets: 52px height on all interactive elements

## Implementation Notes for Executor

1. **globals.css:** Replace all existing CSS variables with the token set in the Color section. Remove the dark mode `@media (prefers-color-scheme: dark)` block entirely — no dark mode in Phase 1.

2. **Font setup:** In `src/app/layout.tsx`, replace Geist imports with `Nunito` and `Nunito_Sans` from `next/font/google`. Load subsets: `['latin']`. Load weights for Nunito: `['700']`. Load weights for Nunito Sans: `['400']`. Only 2 weights total — 400 and 700.

3. **shadcn init:** Run `npx shadcn@latest init`. When asked for style, choose Default. When asked for base color, choose Slate (then override in globals.css with the warm palette — the preset base color doesn't matter since all variables are replaced).

4. **No dark mode:** The generated shadcn `globals.css` will include dark mode variable blocks — delete them. TenniCircle Phase 1 is light-only.

5. **Auth card mobile:** Apply `sm:rounded-2xl sm:shadow-md` pattern — no rounding or shadow at xs, full card treatment at sm+.

6. **Loading spinner:** Use a simple CSS spinner or Lucide `Loader2` icon with `animate-spin` class. Keep it minimal — no third-party loading library.

7. **Token expiry UX (Claude's discretion resolved):** If Supabase returns a 401 mid-session, the proxy.ts will catch it and redirect to /auth with `?session=expired` query param. The auth page checks for this param on load and shows the general error banner: "Your session expired. Please log in again."
